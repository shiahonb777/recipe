Run a recipe in the browser

mlrecipe

The base model and the recipe's LoRA adapter are downloaded into this tab. The merge is done in JavaScript — the same arithmetic peft.merge_and_unload performs in Python — and the result is a complete safetensors file you can save. Nothing is uploaded.

This page only handles base models that fit in browser memory (about 2 GB). For Qwen, Llama, and other large bases, use the CLI: mlrecipe clone … && mlrecipe materialize ./merged.

Examples: shiahonb777/gpt2-alpaca-recipe@v1

Supported bases: gpt2, gpt2-medium, distilgpt2, Qwen/Qwen2.5-0.5B, Qwen/Qwen2.5-0.5B-Instruct. Other recipes require the CLI.

What just happened

  

    The recipe TOML names a base model on Hugging Face and references the LoRA adapter by a
    SHA-256 of its bytes. The page downloaded the base, downloaded the bundle, parsed both as
    safetensors, matched each LoRA (A, B) pair to a base weight, computed
    base + (alpha / rank) · (B · A) in float32, and re-encoded the
    result with the original tensor dtypes preserved. Untouched tensors are byte-identical to
    the base.
  

  

    The Node-side end-to-end test in docs/lib/test_e2e.mjs verifies this against
    PEFT's reference output on the GPT-2 alpaca recipe: 148 of 148 tensors match, max
    element-wise difference 1.19 × 10⁻⁷. That is fp32 round-off; for any meaningful purpose
    the result is identical.
